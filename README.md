# Lunirelust

A clean architecture media-catalog web API built with Axum and SeaORM, with MeiliSearch-powered keyword and semantic (vector) search backed by vLLM embeddings.

## Features

- Clean architecture / DDD with SeaORM for PostgreSQL
- Automated, versioned database migrations (SeaORM CLI)
- JWT authentication and user / device / file management
- RESTful API with OpenAPI docs served at `/docs` (behind the `swagger` cargo feature)
- Unified MeiliSearch index for keyword and hybrid semantic/vector search, with embeddings generated by a vLLM-hosted BGE-M3 model
- Outbox-driven asynchronous indexing and periodic reconciliation between PostgreSQL and MeiliSearch
- Web crawling via the integrated `luneth` crawler with Playwright browser automation

## Tech Stack

- **Backend**: Rust — Axum, SeaORM, tokio, utoipa (OpenAPI)
- **Database**: PostgreSQL
- **Search**: MeiliSearch v1.42 (unified index, `userProvided` embedder)
- **Embeddings**: vLLM serving `BAAI/bge-m3`
- **Crawler**: `luneth` (Playwright), bridged via embedded Node.js
- **Task runner**: `just` (see [`justfile`](justfile))

## Prerequisites

### Rust (stable toolchain)

```bash
rustup default stable
```

### Docker + Docker Compose v2

PostgreSQL, MeiliSearch, and vLLM run in containers. The `justfile` uses the v2 plugin syntax (`docker compose`).

```bash
docker --version
docker compose version
```

### Node.js >= 18

Required by the integrated `luneth` crawler for Playwright-based browser automation. The bridge JS files are embedded in the binary and deployed at runtime.

```bash
node --version  # v18+
```

On Debian/Ubuntu:

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Rust tooling

Install the tools used by the `justfile` and the pre-commit hooks:

```bash
cargo install just                 # task runner
cargo install cargo-watch          # hot reload for `just dev`
cargo install cargo-nextest        # test runner for `just test`
cargo install sea-orm-cli          # migrations / codegen
cargo install --locked cargo-deny  # dependency audit (`just audit`)
cargo install typos-cli            # spell check (`just audit`)
cargo install git-cliff            # changelog generation
```

### Pre-commit

```bash
brew install pre-commit
pre-commit install
```

The hooks enforce repo hygiene, `cargo-fmt`, `cargo-deny`, `cargo-check`, and `cargo-clippy` across the workspace. They do not run the full test suite — use `just test` / `just ci` for that.

## Getting Started

### 1. Clone submodules

```bash
git submodule init
git submodule update
```

### 2. Configure the environment

There is no root `.env.example`; the canonical environment template lives at [`subm/luna/.env.example`](subm/luna/.env.example). Copy it to the repo root as `.env` and adapt the hostnames for local development — the deploy template uses Docker network hostnames (`luna-db`, `luna-meilisearch`), so for local dev switch them to `localhost`:

```bash
cp subm/luna/.env.example .env
```

At minimum the application requires (see `src/common/config.rs`):

- `DATABASE_URL`, `SERVICE_HOST`, `SERVICE_PORT`
- `ASSET_ALLOWED_EXTENSIONS`, `ASSETS_PUBLIC_PATH`, `ASSETS_PUBLIC_URL`, `ASSETS_PRIVATE_PATH`, `ASSETS_PRIVATE_URL`
- Docker Compose vars: `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB`, `POSTGRES_DATA_DIR`, `MEILI_URL`, `MEILI_MASTER_KEY`, `MEILI_PORT`, `MEILI_DATA_DIR`

Optional (have defaults): `CORS_ORIGINS`, `MEILI_*`, `VLLM_EMBEDDING_URL` / `VLLM_EMBEDDING_MODEL` / `VLLM_EMBEDDING_TIMEOUT`, `ASSET_MAX_SIZE`, `DATABASE_MAX_CONNECTIONS`, `DATABASE_MIN_CONNECTIONS`.

## Development

Start the development infrastructure (PostgreSQL + MeiliSearch), apply migrations, and hot-reload the backend with Swagger enabled:

```bash
just dev
```

Stop it:

```bash
just dev-down
```

`just dev` runs the app locally with `cargo watch -x "run -F swagger"`. Once running:

- API base: `http://${SERVICE_HOST}:${SERVICE_PORT}` (set in `.env`)
- Swagger UI / OpenAPI docs: `http://${SERVICE_HOST}:${SERVICE_PORT}/docs`

> The Swagger UI is served only when the `swagger` cargo feature is enabled (as `just dev` does). A plain `cargo run` will not expose `/docs`.

### Full local stack (with vLLM)

For the complete stack used by search indexing and embeddings — the app container together with PostgreSQL, MeiliSearch, and a pinned `vllm/vllm-openai:v0.19.0` container — run:

```bash
just build      # vendor deps, then build and start the full stack
just build-down # stop it
```

The vLLM service is configured with the current pooling-model CLI (`BAAI/bge-m3 --runner pooling`) instead of the removed `--task embed` flag, and caps GPU usage to about `3.6 GiB` on a `6 GiB` card via `--gpu-memory-utilization 0.6`, a shortened maximum sequence length (`--max-model-len 2048`), and reduced batching limits (`--max-num-batched-tokens 2048 --max-num-seqs 32`).

## Database

Apply or roll back migrations with the justfile:

```bash
just migrate-up    # apply pending migrations
just migrate-down  # roll back the last migration
```

For advanced operations, use the `migrate.sh` helper (wraps `cargo run -p migration` / `sea-orm-cli migrate` and loads `.env`):

```bash
./migrate.sh status                     # check migration status
./migrate.sh generate <migration_name>  # scaffold a new migration
./migrate.sh db-reset                   # full reset (removes the container, wipes $POSTGRES_DATA_DIR, re-migrates)
```

## Testing & Quality

```bash
just test-db        # start the test infra (.env.test: postgres + MeiliSearch)
just test-prepare   # test-db + apply migrations (test env)
just test           # default local gate: nextest + doctests
just test-nextest   # nextest only
just test-doc       # doctests only
just test-db-down   # stop the test infra

just check          # cargo check + rustfmt --check + clippy (-D warnings)
just ci             # full CI: check + test
just audit          # cargo-deny + typos
```

The search integration tests require a live MeiliSearch instance, which `just test` / `just test-db` starts from `.env.test`. They auto-skip when MeiliSearch is unreachable, so a bare `cargo nextest run` without the compose stack does not fail.

### Pre-commit

The hooks run formatting, `cargo-deny`, `cargo-check`, and `cargo-clippy` — not the full test suite. Use `just test` / `just ci` for the complete gate:

```bash
pre-commit run --all-files
```

## Architecture

Clean architecture organized by domain:

- `src/entities/` — SeaORM entities (generated from migrations)
- `src/domains/` — business domains: `auth`, `user`, `device`, `file`, `luna` (media catalog), `search` (MeiliSearch indexing + search), `crawl` (`luneth` integration)
- `src/common/` — shared configuration, JWT, error handling, pagination
- `migration/` — SeaORM database migrations
- `subm/luna/` — runtime environment (docker-compose, data volumes, deploy scripts)
- `subm/lust/` — standalone CLI tool for ID extraction / dedup
